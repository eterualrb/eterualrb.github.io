git查看文件修改纪录&&可视化工具

2016.08.23 王白告  热度 ℃ 条评论

1. 查看文件历史修改纪录

git blame

运行git blame [filename]可以得到整个文件每一行的详细修改信息，包括SHA1串，日期和作者，加”-L”参数可以查看指定开始和结束行内的文件修改纪录。

git show

显示文件最新改变的具体内容(git show还有其他功能，比如查看tag信息)，一般配合commit的sha1值使用，详细内容见git log部分

2. 查看仓库历史修改纪录

git log

git log –name-status
每次修改的文件列表及状态

git log –name-only

每次修改的文件列表

git log –stat

每次修改的文件列表, 及文件修改的统计

git whatchanged

git whatchanged命令与git log命令类似

  • ＃git whatchanged 每次修改的文件列表
  • ＃git whatchanged –stat 每次修改的文件列表, 及文件修改的统计

git log配合git show查看某个文件的历史修改纪录

获取到某次提交的sha1值后，再使用git show就可以查看到这次提交中该文件的具体修改内

注意：git show 并不是用来显示某个文件的信息，而是显示某次commit的具体信息
配合git log –pretty=oneline也只是打印出某次commit的信息
查看某个文件的修改内容还需要在这次commit中去查找

3. 图形化工具查看修改纪录

AS自带版本控制工具

as自带的功能其实已经很强大了，其它可视化工具该有的功能基本都已经包含了

mac平台主流图形化工具source tree

ubuntu平台主流图形化工具gitk和gitg

gitk和gitg是ubuntu上用的最多的图形化工具，常配合文件差分工具一起使用，功能强大
gitg还可以查看stash中的内容
但是在mac上用的感觉像是阉割版，好多功能不支持了，而且安装不方便，对mac的retina显示屏支持不好
